What is the difference between classes and IDs in CSS?

- Classes are used to style a group of elements that are given a specific class tag in HTML.
- IDs are used for styling a specific element.

What is the difference between block, inline, and inline-block?

Block Elements:
  • respect left, right, top and bottom margins and padding
  • force a line break after each block element

Inline Elements:
  • respect left and right margins and padding but not top and bottom
  • cannot have a width and height set
  • allow other elements to sit to their left and right

Inline-block Elements:
  • allow other elements to sit to their left and right
  • respect top and bottom margins and padding
  • respect height and width

What's the difference between a relative, fixed, absolute and statically positioned element?

Static (default)
  • not affected by the top, bottom, left, and right properties
  • always positioned according to the normal flow of the page

Relative
  • This type of positioning needs to be complemented with some sort of positioning attribute. Otherwise, it will stay as if the element is statically positioned.
  • If you do have some other positioning attribute such as top: 10px, it will move the elements position DOWN 10px.
  • Elemenets that are positioned relatively have the ability to use the z-index on that element.

Fixed
  • A fixed element stays where it is relative to the viewpoint (browser window itself)
  • For example, a fixed element will stay where it is even when the page is scrolled

Absolute
  • This type of positioning allows you put the element where ever you wish by using the positioning attributes top, left, bottom, and right
  • This element is bound to its next parent so these positioning attribute values will be positioned relative to its next parent with relative or absolute positioning. If no such parent exists, this element will be positioned relative to the entire page itself.
  • Elements that are positioned absolutely are removed from the flow of the page and are not affected by other elements. These elements also do not affect any other elements on the page.

What is the difference between "resetting" and "normalizing" in CSS? Which would you choose and why?

- Reset will remove all default styling applied by the browser giving you a blank canvas.
- Normalize will give you a base stylesheet and styles default elements to be consistent across various different browsers.
- Personally, I would use normalize.css more because it has become more of an industry standard at this point. Normalize.css is also made with HTML5 in mind which makes it all that easier to keep up with modern standards in web development.

What is a float and when would you use it?

- "Float" is a CSS positioning property that helps images properly wrap around text. Floats can be set to right or left which positions the element to wrap around the text in that respective position.

Describe z-index and how stacking context is formed.

- The z-index determines the order in which different elements are vertically stacked.
- It only affects elements that have a position value other than static (the default).
- Without the z-index, the elements vertically stack as they appear in the DOM.

Describe Block Formattting Context (BFC) and how it works.

- A BFC is defined as an HTML box where
  • float : none
  • position : fixed | absolute
  • display : table-cell | table-caption | inline-block | inline-flex
  • overflow : hidden

What are the various clearing techniques and which is appropriate for what context?

- Clearing floats forces the containing element to expand to contain its child elements. It essentially forces its child elements to appear below it.
- There are four ways to clear floats.
  • .clear {clear: both;}
  • .container{overflow:hidden;}
  • .clearfix uses the pseudo-elements ::before and ::after. The ::after element is used to clear the floats.
  • (the newest way).container{min-height: contain-floats;}

Explain CSS sprites and how you would implement them.

- CSS Sprites are basically taking multiple images, making them into one image, and displaying only part of that large image at a time.
- Although the total image size goes up, several images are able to load with a single HTTP request.
- The traditional way of using CSS sprites is setting the same background-image on several CSS classes and using the background-position and dimensions of the individual classes to display a single portion of the sprite.
- Another way to use CSS Sprites is to use several frameworks such as Node.js, Compass, Grunt, Gulp, etc.

Image Replacement Techniques

CSS Image Replacement is a technique of replacing a text element (usually a header tag) with an image. The most common usage of this technique is to display a logo on a page.
Phark Method
  • - This method is used by replacing the block with a background image and kicking the text off the page with an indedent.
  • - This is the most widely used technique today and keeps screen readers happy

FIR (Fahrner Image Replacement)
  • - FIR is a technique that uses a span to wrap the text inside the header and use the span to hide the text.
  • - Since we use display: none property, this hides the text from screen readers and search bots
